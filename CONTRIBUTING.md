# Contributing

Thank you for your interest in improving this project.

## Before You Start

- Open an issue to discuss significant changes before implementation.
- Keep pull requests focused and small.
- Follow existing style and structure in the repository.

## Development Setup

1. Create and activate a virtual environment.
2. Install dependencies from requirements.txt.
3. Copy .env.example to .env.
4. Run the application locally and verify your change.

## Pull Request Checklist

- Update docs when behavior or setup changes.
- Add or update tests when practical.
- Avoid committing secrets, local logs, model caches, or virtualenv files.
- Confirm the project still runs with the documented commands.

## Commit Guidance

- Use clear, descriptive commit messages.
- Keep one logical change per commit when possible.

## Code of Conduct

By participating in this project, you agree to collaborate respectfully and constructively.
